# ADR Guide

## What An ADR Is

An ADR records a significant architectural decision, the context around it, and its consequences.

## ADR Template

- Title
- Status
- Context
- Decision
- Consequences
- Alternatives Considered

## Status Values

- Proposed
- Accepted
- Superseded
- Deprecated

## When To Write An ADR

- When a decision changes subsystem boundaries
- When a decision changes source-of-truth semantics
- When a decision changes security or signing behavior
- When a decision introduces or removes a major external dependency

## Relationship To Main Architecture Docs

Main architecture docs explain the system as it stands or is intended to stand. ADRs explain why important choices were made.
